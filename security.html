Top 5 tips on protecting yourself online

  1. Use effective passwords as much as possible and do not use the same password.

    2. Try to use letters, numbers and symbols in your passwords to make them harder to guess.

  2. Use antivirus programs to protect yourself from viruses.

    3. There are many antivirus programs, some are paid some are free.

  3. Turn off third party cookies.

    4. This is how advertisers access information about you.

  4. Do not share personal information on the internet.

    5. This includes your name, number, banking details and address.

  5. Try to stay away from public networks.

    6. You don't know who's watching you on public networks.

What responsibilities do developers have in data security?

When getting data from users and sending it, web developers in particular need to use POST requests in HTML forms when manipulating user data as opposed to GET requests. GET requests should never be used in regards to getting and sending personal data to servers. POST requests are much safer than GET requests and protect the user data better.

GET requests POST requests
Can be cached. Are never cached.
Remain in browser history. Do not remain in browser history.
Display personal data in URL. Do not display personal data in URL.
Can be bookmarked. Cannot be bookmarked.
Have length restrictions. Do not have length restrictions.

Web application developers should also use asterisks when displaying passwords instead of the actual letters, numbers and symbols. When storing passwords in server databases, an algorithm should be used to turn passwords into hashes when storing to increase security. There should also be an algorithm for reversing the hash back into the original password when needed.
